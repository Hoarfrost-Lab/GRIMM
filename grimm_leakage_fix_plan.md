## Workstream 3 — clean v2 dataset (fresh GRIMM-scoped regeneration)

Fresh regen at **50% identity reusing existing UniRef50 IDs** (no re-clustering, no TEACUP/extensions scope changes). Borrow the *algorithm* from `~/Projects/TEACUP/src/teacup/splitting_engine.py` (by-cluster, shuffled folds, seeded, QC) but keep **original GRIMM scope/conventions** — notably **do NOT expand multi-EC rows** (keep compound EC labels as v1 did), so v2 is a faithful fixed-v1, not a re-scoped dataset.

Fixes by construction: A (correct test2), B (split by cluster), D (shuffle cluster→partition per fold), E (nuc per-file), F (seeded).

**Data acquisition (none of the source files are local):**
- Reconstruct the base table (Entry, EC number, EMBL, RefSeq, UniRef50/90/100) from the **union of v1 deployed partitions** → preserves "existing UniRef50 IDs" with no download.
- **[OPEN — Bug C] AA sequences:** real per-protein sequences require downloading `uniprot_sprot.fasta` (pinned to the 2025-05-23 release) and mapping Entry→sequence. Alternative: keep UniRef50 representative sequences (no download), which is leakage-free once B/D are fixed but retains the "duplicate input, possibly different label" artifact.
- Nucleotides: reconstruct Entry→CDS mapping from v1 nucleotide partitions, re-split with the fixed nuc logic.

**QC:** cluster-exclusivity per label; 0 leakage across all partitions by accession & sequence; confirm folds are independent.

**Hosting:** same HF repo `HoarfrostLab/GRIMM`, under a versioned path (e.g. `EC_v2/...` alongside the existing `EC/...`).
