## Key reframing ‚Äî B and C are deviations from the PAPER, not design choices

The preprint (arXiv 2602.16504v1) documents the *correct* method: per-protein **SwissProt sequences** (UniRef50 used only for stratification), 2-cluster labels split **by cluster** (1 train / 1 test-1), singleton-cluster labels are orphans. The **released data does not implement this**:
- AA `Sequence` is the **UniRef50 representative** (verified live: distinct clusters == distinct sequences == 48,966; one 196-accession cluster has 1 sequence) ‚Äî Bug C.
- Low-support handling keys on **row count** and splits **by row**, not by cluster ‚Äî Bug B.

So fixing B + C in v2 = making the data finally match the paper. **Both submitted papers used the AA splits**, so they inherit both issues ‚Üí v1 README must disclose them and steer users to v2.
